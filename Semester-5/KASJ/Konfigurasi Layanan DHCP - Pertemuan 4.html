DHCP Process - DORA
Discovery • Offer • Request • Acknowledgment
1

DISCOVER

Client broadcasts request untuk IP address

UDP Port 67
→
2

OFFER

Server offers available IP address

UDP Port 68
→
3

REQUEST

Client requests offered IP address

Broadcast Message
→
4

ACK

Server acknowledges dan assigns IP

Lease Duration
Important Notes:
• DHCP menggunakan UDP protocol (connectionless)
• Broadcast pada network segment yang sama
• Lease time menentukan durasi assignment
• Client akan renew lease pada 50% dari lease time

PowerShell Method

# Install DHCP Server Feature
Install-WindowsFeature -Name DHCP `
    -IncludeManagementTools

# Add DHCP Server to Active Directory
Add-DhcpServerInDC -DnsName "server.domain.com" `
    -IPAddress 192.168.1.10

# Set DHCP Server credentials
Set-DhcpServerDnsCredential -Credential `
    (Get-Credential) -ComputerName "server"
Verification:
Get-WindowsFeature -Name DHCP
Linux DHCP Server Installation

Ubuntu/Debian Installation

# Update package repository
sudo apt update

# Install ISC DHCP Server
sudo apt install isc-dhcp-server

# Enable and start service
sudo systemctl enable isc-dhcp-server
sudo systemctl start isc-dhcp-server
Note: Service will fail first time until configuration is completed

CentOS/RHEL Installation

# Install DHCP Server package
sudo yum install dhcp-server
# or for newer versions
sudo dnf install dhcp-server

# Enable service
sudo systemctl enable dhcpd
sudo systemctl start dhcpd

# Check service status
sudo systemctl status dhcpd
Config Files:
• Ubuntu: /etc/dhcp/dhcpd.conf
• CentOS: /etc/dhcp/dhcpd.conf
DHCP Scope Configuration
IP Address Pool Management

Windows Server Scope

Scope Properties:
• Name: LAN Scope
• Start IP: 192.168.1.100
• End IP: 192.168.1.200
• Subnet Mask: 255.255.255.0
• Lease Duration: 8 days
• Router: 192.168.1.1
• DNS: 8.8.8.8, 8.8.4.4 
# PowerShell Scope Creation
Add-DhcpServerv4Scope -Name "LAN Scope" `
    -StartRange 192.168.1.100 `
    -EndRange 192.168.1.200 `
    -SubnetMask 255.255.255.0 `
    -LeaseDuration 8.00:00:00

Linux DHCP Configuration

# /etc/dhcp/dhcpd.conf
default-lease-time 691200;  # 8 days
max-lease-time 691200;

subnet 192.168.1.0 netmask 255.255.255.0 {
    range 192.168.1.100 192.168.1.200;
    option routers 192.168.1.1;
    option domain-name-servers 8.8.8.8, 8.8.4.4;
    option domain-name "local.domain";
    option broadcast-address 192.168.1.255;
}
Restart Service:
sudo systemctl restart isc-dhcp-server

# PowerShell Reservation
Add-DhcpServerv4Reservation `
    -ScopeId 192.168.1.0 `
    -IPAddress 192.168.1.50 `
    -ClientId "00-15-5D-00-27-01" `
    -Name "File Server"

Linux DHCP Reservations

# /etc/dhcp/dhcpd.conf
# Fixed IP reservations
host fileserver {
    hardware ethernet 00:15:5d:00:27:01;
    fixed-address 192.168.1.50;
    option host-name "fileserver";
}

host printer {
    hardware ethernet 08:00:27:12:34:56;
    fixed-address 192.168.1.60;
    option host-name "printer";
}

host webserver {
    hardware ethernet 52:54:00:12:34:56;
    fixed-address 192.168.1.70;
}
Best Practice: Always use reservations for servers, printers, dan critical devices
DHCP Options
Additional Network Parameters

Common Options

Option 3: Router/Gateway
Option 6: DNS Servers
Option 15: Domain Name
Option 42: NTP Servers
Option 44: WINS Servers

Windows Configuration

# Set DHCP Options via PowerShell
Set-DhcpServerv4OptionValue `
    -OptionId 3 `
    -Value 192.168.1.1

Set-DhcpServerv4OptionValue `
    -OptionId 6 `
    -Value 8.8.8.8,8.8.4.4

Set-DhcpServerv4OptionValue `
    -OptionId 15 `
    -Value "company.local"

Linux Configuration

# /etc/dhcp/dhcpd.conf
option routers 192.168.1.1;
option domain-name-servers 
    8.8.8.8, 8.8.4.4;
option domain-name "company.local";
option ntp-servers 192.168.1.5;
option netbios-name-servers 
    192.168.1.10;
Option Hierarchy: • Server Options: Applied to all scopes globally
• Scope Options: Applied to specific scope only
• Reservation Options: Applied to specific client only

DHCP Failover
High Availability Configuration

Windows Server Failover

1
Configure Primary Server
Setup DHCP scopes dan options
2
Install Secondary Server
Install DHCP role on second server
3
Configure Failover Relationship
Right-click scope → Configure Failover 
# PowerShell Failover Setup
Add-DhcpServerv4Failover `
    -Name "DHCP-Failover" `
    -PartnerServer "DHCP-Server2" `
    -ScopeId 192.168.1.0 `
    -LoadBalancePercent 50

Linux DHCP Failover

# /etc/dhcp/dhcpd.conf (Primary Server)
failover peer "dhcp-failover" {
    primary;
    address 192.168.1.10;
    port 647;
    peer address 192.168.1.11;
    peer port 647;
    max-response-delay 60;
    max-unacked-updates 10;
    split 128;
    mclt 3600;
    load balance max seconds 3;
}

# Pool configuration with failover
pool {
    failover peer "dhcp-failover";
    range 192.168.1.100 192.168.1.200;
}
Failover Modes: Load Balance (50/50) atau Hot Standby (active/passive)
DHCP Testing & Troubleshooting

Testing Tools

# Windows Client Testing
ipconfig /release
ipconfig /renew
ipconfig /displaydns

# DHCP Server testing
Get-DhcpServerv4Lease -ScopeId 192.168.1.0
Get-DhcpServerv4Statistics

# Network testing
ping dhcp-server-ip
telnet dhcp-server-ip 67
# Linux Client Testing
sudo dhclient -r eth0    # Release
sudo dhclient eth0       # Renew

# DHCP server testing
sudo journalctl -u isc-dhcp-server
tail -f /var/log/syslog | grep dhcp

Common Issues

Problem Solution
No IP Address Check scope activation, server status
Wrong IP Range Verify scope configuration
No DNS Resolution Check DHCP option 6 configuration
Reservation Not Working Verify MAC address format
Cross-Subnet Issues Configure DHCP relay
DHCP Monitoring & Logs

Windows Server Monitoring

# Enable DHCP Logging
netsh dhcp server set optionvalue 81 BYTE 1

# View DHCP Statistics
Get-DhcpServerv4Statistics
Get-DhcpServerv4ScopeStatistics

# Check Event Logs
Get-WinEvent -LogName "Microsoft-Windows-Dhcp-Server/Operational"

# Monitor lease database
Get-DhcpServerv4Lease -AllLeases
Log Location:
%SystemRoot%\System32\Dhcp\

Linux Server Monitoring

# View DHCP logs
sudo tail -f /var/log/syslog | grep dhcp
sudo journalctl -u isc-dhcp-server -f

# Check lease file
sudo cat /var/lib/dhcp/dhcpd.leases

# Monitor active leases
sudo dhcp-lease-list

# Server status
sudo systemctl status isc-dhcp-server
# Custom monitoring script
#!/bin/bash
echo "DHCP Server Status:"
systemctl is-active isc-dhcp-server
echo "Active leases:"
grep "binding state active" /var/lib/dhcp/dhcpd.leases | wc -l
