Domain Name System (DNS)
The Internet's Phone Book
DNS = Name Resolution Service yang menerjemahkan domain names ke IP addresses

Fungsi Utama

Name Resolution
Domain → IP Address
Reverse Lookup
IP Address → Domain

Keuntungan

Human Readable
Easy to remember names
Flexibility
IP can change transparently

Use Cases

Web Browsing
www.example.com
Email Services
MX record resolution
DNS Hierarchy Structure
Distributed Database System
Root Level (".")
13 Root Servers Worldwide
↓
Top Level Domain (TLD)
.com, .org, .net, .id, .edu
↓
Second Level Domain
google, facebook, untidar
↓
Subdomain
www, mail, ftp, api
Example Resolution: www.untidar.ac.id
• Root → .id (Indonesia TLD) → ac.id (Academic) → untidar.ac.id → www.untidar.ac.id
DNS Resolution Process
Step-by-Step Query Resolution

Recursive Resolution

1
Client Query
User types www.example.com
2
Local Cache Check
Check resolver cache
3
Recursive Server
Query DNS server
4
Root Server Query
Ask root for .com TLD

Iterative Process

5
TLD Server Query
Ask .com for example.com
6
Authoritative Server
Query example.com DNS
7
Answer Retrieved
Get A record with IP
8
Response to Client
Return IP to client

BIND Forward Zone

# /etc/bind/named.conf.local
zone "contoh.local" {
    type master;
    file "/etc/bind/db.contoh.local";
};
; /etc/bind/db.contoh.local
$TTL    604800
@       IN      SOA     ns1.contoh.local. admin.contoh.local. (
                        2023010101      ; Serial
                        604800          ; Refresh
                        86400           ; Retry
                        2419200         ; Expire
                        604800 )        ; Minimum
;
@       IN      NS      ns1.contoh.local.
@       IN      A       192.168.1.10
ns1     IN      A       192.168.1.10
www     IN      A       192.168.1.20
mail    IN      A       192.168.1.30
ftp     IN      CNAME   www.contoh.local.

Network Format:

BIND Reverse Zone

# /etc/bind/named.conf.local
zone "1.168.192.in-addr.arpa" {
    type master;
    file "/etc/bind/db.192.168.1";
};
; /etc/bind/db.192.168.1
$TTL    604800
@       IN      SOA     ns1.contoh.local. admin.contoh.local. (
                        2023010101      ; Serial
                        604800          ; Refresh
                        86400           ; Retry
                        2419200         ; Expire
                        604800 )        ; Minimum
;
@       IN      NS      ns1.contoh.local.
10      IN      PTR     ns1.contoh.local.
20      IN      PTR     www.contoh.local.
30      IN      PTR     mail.contoh.local.

Windows DNS Forwarders

1

Right-click server → Properties
2
Forwarders Tab
Add forwarder IP addresses
3
Common Forwarders
8.8.8.8, 1.1.1.1, 208.67.222.222 
# PowerShell Configuration
Add-DnsServerForwarder -IPAddress "8.8.8.8"
Add-DnsServerForwarder -IPAddress "1.1.1.1"

# Verify forwarders
Get-DnsServerForwarder

BIND Forwarders

# /etc/bind/named.conf.options
options {
    directory "/var/cache/bind";
    
    forwarders {
        8.8.8.8;
        1.1.1.1;
        208.67.222.222;
    };
    
    forward only;
    
    dnssec-validation auto;
    listen-on-v6 { any; };
};

DNS Testing & Troubleshooting Tools

Windows Tools

# nslookup - Interactive DNS lookup
nslookup www.example.com
nslookup 192.168.1.20

# dig equivalent for Windows
Resolve-DnsName www.example.com
Resolve-DnsName -Type MX example.com

# Test DNS server
Test-DnsServer -IPAddress 8.8.8.8

# Clear DNS cache
Clear-DnsClientCache
ipconfig /flushdns

Linux Tools

# dig - Domain Information Groper
dig www.example.com
dig @8.8.8.8 example.com MX
dig -x 192.168.1.20

# nslookup
nslookup www.example.com
nslookup www.example.com 8.8.8.8

# host command
host www.example.com
host 192.168.1.20

# Check BIND configuration
named-checkconf /etc/bind/named.conf
named-checkzone example.com /etc/bind/db.example.com
Testing Checklist:
✓ Forward lookup (A records) • ✓ Reverse lookup (PTR records) • ✓ MX records for email • ✓ NS records for delegation • ✓ Zone transfer functionality • ✓ Forwarder configuration
Common DNS Issues & Solutions
Issue Symptoms Solution
No DNS Resolution Cannot resolve any domain names Check DNS service, network connectivity, forwarders
Slow Resolution DNS queries take long time Check forwarders, network latency, cache settings
Reverse Lookup Fails IP to name resolution not working Check PTR records, reverse zone configuration
Zone Transfer Errors Secondary server not updating Check zone transfer settings, firewall, SOA serial
Cache Poisoning Wrong IP addresses returned Clear DNS cache, check security settings
NXDOMAIN Errors Domain not found responses Check zone files, record syntax, delegation
