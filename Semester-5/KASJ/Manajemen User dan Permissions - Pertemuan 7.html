Linux User Management

User Management Commands

# Create new user
sudo useradd -m -s /bin/bash username
sudo useradd -m -G sudo,www-data john

# Set password
sudo passwd username

# Modify user
sudo usermod -aG groupname username
sudo usermod -l newname oldname

# Delete user
sudo userdel username
sudo userdel -r username  # with home directory
User Info Files:
/etc/passwd, /etc/shadow, /etc/group

Group Management

# Create group
sudo groupadd groupname

# Add user to group
sudo usermod -aG groupname username
sudo gpasswd -a username groupname

# Remove user from group
sudo gpasswd -d username groupname

# List user groups
groups username
id username

# Delete group
sudo groupdel groupname
Important Groups:
sudo, wheel, www-data, docker
Windows User Management

Command Line Management

# Create new user
net user username password /add
net user john P@ssw0rd! /add /fullname:"John Doe"

# Add to group
net localgroup "Administrators" username /add
net localgroup "Remote Desktop Users" john /add

# Modify user
net user username /active:yes
net user username /expires:never

# Delete user
net user username /delete

PowerShell Commands

# Create user
New-LocalUser -Name "john" -Password (ConvertTo-SecureString "P@ssw0rd!" -AsPlainText -Force) -FullName "John Doe"

# Add to group
Add-LocalGroupMember -Group "Administrators" -Member "john"

# List users
Get-LocalUser

# Modify user
Set-LocalUser -Name "john" -Description "IT Admin"

# Remove user
Remove-LocalUser -Name "john"
Security Note: Always use strong passwords and follow company password policy

Role-Based Access Control (RBAC)
Organizational Permission Management

Administrator

Full system access, user management, security policies

Developer

Application deployment, development tools, code repositories

Analyst

Data access, reporting tools, read-only system info

Manager

Team resources, project folders, approval workflows

End User

Personal files, shared resources, basic applications

Guest

Limited access, temporary resources, restricted permissions

RBAC Benefits: Simplified management • Consistent permissions • Easier auditing • Reduced errors • Scalable security model

Password Security Policies

Linux Password Policy

# Install password quality tools
sudo apt install libpam-pwquality

# Configure /etc/security/pwquality.conf
minlen = 12
minclass = 3
maxrepeat = 2
dcredit = -1
ucredit = -1
lcredit = -1
ocredit = -1

# Password aging in /etc/login.defs
PASS_MAX_DAYS 90
PASS_MIN_DAYS 1
PASS_WARN_AGE 7
Enforce for existing users:
sudo chage -M 90 -m 1 -W 7 username

Windows Password Policy

Minimum Length: 12 characters
Maximum Age: 90 days
History: Remember last 12 passwords
Lockout: 5 failed attempts = 30 min lockout 
# PowerShell commands
Get-ADDefaultDomainPasswordPolicy
Set-ADDefaultDomainPasswordPolicy -MinPasswordLength 12 -PasswordHistoryCount 12
File Security Implementation
Practical Scenarios

Scenario: Departmental File Structure

Requirement: Create secure file structure untuk HR, Finance, dan IT departments dengan appropriate access controls.

Linux Implementation

# Create directory structure
sudo mkdir -p /company/{hr,finance,it,shared}

# Create groups
sudo groupadd hr-dept
sudo groupadd finance-dept  
sudo groupadd it-dept

# Set permissions
sudo chown root:hr-dept /company/hr
sudo chmod 2770 /company/hr

sudo chown root:finance-dept /company/finance
sudo chmod 2770 /company/finance

sudo chown root:it-dept /company/it
sudo chmod 2770 /company/it

# Shared folder - all departments read access
sudo chown root:root /company/shared
sudo chmod 755 /company/shared

Windows Implementation

# PowerShell script
$departments = @("HR", "Finance", "IT")

foreach ($dept in $departments) {
    # Create directory
    New-Item -Path "C:\Company\$dept" -ItemType Directory -Force
    
    # Create group
    New-LocalGroup -Name "$dept-Dept" -Description "$dept Department"
    
    # Set permissions
    $acl = Get-Acl "C:\Company\$dept"
    $accessRule = New-Object System.Security.AccessControl.FileSystemAccessRule("$dept-Dept","FullControl","ContainerInherit,ObjectInherit","None","Allow")
    $acl.SetAccessRule($accessRule)
    Set-Acl "C:\Company\$dept" $acl
}
Advanced Permission Concepts

Linux ACLs (Access Control Lists)

# Install ACL support
sudo apt install acl

# Mount with ACL support
sudo mount -o remount,acl /

# Set ACL permissions
setfacl -m user:john:rwx /path/to/file
setfacl -m group:developers:rw /path/to/directory

# View ACL
getfacl /path/to/file

# Remove ACL
setfacl -x user:john /path/to/file

# Default ACLs for directories
setfacl -d -m group:developers:rw /path/to/directory

Auditing and Monitoring
Compliance and Security Tracking

Linux Audit System

# Install auditd
sudo apt install auditd

# Configure audit rules
sudo nano /etc/audit/rules.d/audit.rules

# Monitor file access
-w /etc/passwd -p wa -k passwd_changes
-w /etc/shadow -p wa -k shadow_changes
-w /company/hr -p rwxa -k hr_access

# Start audit service
sudo systemctl start auditd
sudo systemctl enable auditd

# Search audit logs
sudo ausearch -k passwd_changes
sudo aureport --summary

Windows Audit Policy

# Enable audit policies
auditpol /set /subcategory:"Logon" /success:enable /failure:enable
auditpol /set /subcategory:"File System" /success:enable /failure:enable

# PowerShell audit configuration
Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\Lsa" -Name "AuditBaseObjects" -Value 1

# View audit events
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4624}  # Successful logon
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625}  # Failed logon
