Konsep Firewall dan Access Control

Apa itu Firewall?

Network security system yang mengontrol traffic berdasarkan predetermined security rules

Jenis Firewall

1
Packet Filtering
Inspects packets berdasarkan IP, port, protocol
2
Stateful Inspection
Tracks connection state dan context
3
Application Layer
Deep packet inspection pada application data
4
Next-Gen Firewall
Combines traditional firewall dengan advanced features

Security Policies

Default Deny Policy

Block semua traffic kecuali yang explicitly allowed

Least Privilege

Grant minimum access yang diperlukan untuk operasi

Defense in Depth

Multiple layers of security controls

Best Practices

• Regular rule review dan cleanup

• Logging dan monitoring semua activities

• Change management process

• Regular security assessments

Konfigurasi Windows Firewall

Praktik: Windows Firewall with Advanced Security

Basic Configuration

# Enable Windows Firewall (PowerShell) Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled True # Check firewall status Get-NetFirewallProfile # View existing rules Get-NetFirewallRule | Where-Object {$_.Enabled -eq 'True'}
1
Access Advanced Security
Windows + R → wf.msc → Advanced Settings
2
Configure Profiles
Domain, Private, Public network profiles

Firewall Rules

Inbound Rule Example: Rule Name: Allow HTTP Action: Allow Protocol: TCP Local Port: 80 Remote Address: Any Scope: All Profiles
# Create firewall rule (PowerShell) New-NetFirewallRule -DisplayName "Allow SSH" ` -Direction Inbound ` -Protocol TCP ` -LocalPort 22 ` -Action Allow # Block specific IP New-NetFirewallRule -DisplayName "Block Malicious IP" ` -Direction Inbound ` -RemoteAddress 192.168.1.100 ` -Action Block

Security Notes

• Test rules sebelum production deployment

• Document semua custom rules

• Regular audit dan cleanup unused rules

Konfigurasi Linux iptables

Praktik: iptables Firewall Setup

Basic iptables Commands

# View current rules
sudo iptables -L -n -v

# Flush all rules
sudo iptables -F

# Set default policies
sudo iptables -P INPUT DROP
sudo iptables -P FORWARD DROP
sudo iptables -P OUTPUT ACCEPT

# Allow loopback
sudo iptables -A INPUT -i lo -j ACCEPT
sudo iptables -A OUTPUT -o lo -j ACCEPT
Common Rules: # Allow SSH iptables -A INPUT -p tcp --dport 22 -j ACCEPT # Allow HTTP/HTTPS iptables -A INPUT -p tcp --dport 80 -j ACCEPT iptables -A INPUT -p tcp --dport 443 -j ACCEPT # Allow established connections iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

Advanced iptables Rules

# Rate limiting (DDoS protection)
iptables -A INPUT -p tcp --dport 80 -m limit --limit 25/minute --limit-burst 100 -j ACCEPT

# Block specific countries (using GeoIP)
iptables -A INPUT -m geoip --src-cc CN,RU -j DROP

# Log dropped packets
iptables -A INPUT -j LOG --log-prefix "DROPPED: "
iptables -A INPUT -j DROP

# Save rules (Ubuntu/Debian)
sudo iptables-save > /etc/iptables/rules.v4

firewalld (CentOS/RHEL)

# Basic firewalld commands
sudo firewall-cmd --get-active-zones
sudo firewall-cmd --list-all

# Add service
sudo firewall-cmd --add-service=http --permanent
sudo firewall-cmd --reload

# Add port
sudo firewall-cmd --add-port=8080/tcp --permanent
Access Control Lists (ACLs)

Network ACL Concepts

Standard ACLs

Filter berdasarkan source IP address only

Extended ACLs

Filter berdasarkan source/destination IP, ports, protocols

Cisco Router ACL Example: # Standard ACL access-list 10 permit 192.168.1.0 0.0.0.255 access-list 10 deny any # Extended ACL access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 80 access-list 100 permit tcp 192.168.1.0 0.0.0.255 any eq 443 access-list 100 deny ip any any # Apply to interface interface FastEthernet0/1 ip access-group 100 in

Linux ACL Implementation

# Install ACL tools
sudo apt-get install acl

# Set file ACL
setfacl -m u:username:rwx /path/to/file
setfacl -m g:groupname:r-x /path/to/directory

# View ACLs
getfacl /path/to/file

# Remove ACLs
setfacl -x u:username /path/to/file
setfacl -b /path/to/file  # Remove all ACLs

Network Segmentation

Gunakan ACLs untuk:

• Isolate sensitive networks

• Control inter-VLAN communication

• Implement DMZ policies

• Restrict administrative access

ACL Best Practices

• Place extended ACLs close to source

• Place standard ACLs close to destination

• Use descriptive names/comments

• Regular review dan cleanup

Intrusion Detection Systems (IDS)

IDS/IPS Implementation & Management

IDS vs IPS

IDS (Detection)

• Monitors dan alerts on suspicious activities

• Passive monitoring (out-of-band)

• Historical analysis capabilities

• No impact on network performance

IPS (Prevention)

• Active blocking of malicious traffic

• Inline deployment (in-band)

• Real-time threat mitigation

• Potential performance impact

1
Network-based IDS/IPS
Monitors network traffic dan protocols
2
Host-based IDS/IPS
Monitors individual host activities

Detection Methods

Signature-based Detection

• Matches known attack patterns

• Low false positives

• Cannot detect zero-day attacks

Anomaly-based Detection

• Detects deviations from normal behavior

• Can detect unknown attacks

• Higher false positive rates

Common IDS Alerts

PORT SCAN DETECTED
SQL INJECTION ATTEMPT
UNUSUAL TRAFFIC PATTERN
FAILED LOGIN ATTEMPTS
POLICY VIOLATION
# Snort IDS Configuration Example
# /etc/snort/snort.conf
var HOME_NET 192.168.1.0/24
var EXTERNAL_NET !$HOME_NET

# Sample rule
alert tcp $EXTERNAL_NET any -> $HOME_NET 22 (msg:"SSH Connection Attempt"; sid:1000001;)
