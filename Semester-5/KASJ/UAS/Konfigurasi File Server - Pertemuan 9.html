File Server Fundamentals

Apa itu File Server?

File Server adalah server yang dirancang khusus untuk menyimpan, mengelola, dan menyediakan akses ke files melalui jaringan.

Fungsi Utama

Centralized Storage - Penyimpanan terpusat
Multi-user Access - Akses bersamaan
Access Control - Kontrol permission
Data Synchronization - Sinkronisasi data

Protocol File Sharing

SMB
SMB/CIFS (Samba)
Protocol untuk Windows networking, cross-platform compatible
NFS
Network File System
Native Unix/Linux file sharing protocol
FTP
File Transfer Protocol
Transfer files over network, SFTP untuk security
SSH
SSH/SCP
Secure file transfer menggunakan SSH protocol
Setup Samba File Server

Praktik: Konfigurasi Samba Server

Installation & Basic Setup

# Install Samba
sudo apt update
sudo apt install samba samba-common-bin

# Check status
sudo systemctl status smbd
sudo systemctl enable smbd
1
Create Share Directory
sudo mkdir -p /srv/samba/shared
sudo chmod 777 /srv/samba/shared
2
Add Samba Users
sudo smbpasswd -a username
sudo smbpasswd -e username

Samba Configuration

# /etc/samba/smb.conf
[global]
    workgroup = WORKGROUP
    server string = File Server
    security = user
    map to guest = bad user
    dns proxy = no

[shared]
    comment = Shared Directory
    path = /srv/samba/shared
    browseable = yes
    read only = no
    guest ok = no
    create mask = 0755
    directory mask = 0755
# Test configuration
sudo testparm

# Restart services
sudo systemctl restart smbd nmbd
Setup NFS File Server

Praktik: Konfigurasi NFS Server

NFS Server Setup

# Install NFS
sudo apt install nfs-kernel-server

# Create NFS directory
sudo mkdir -p /srv/nfs/public
sudo chown nobody:nogroup /srv/nfs/public
sudo chmod 755 /srv/nfs/public
# /etc/exports
/srv/nfs/public    192.168.1.0/24(rw,sync,no_subtree_check)
/home               192.168.1.0/24(rw,sync,no_root_squash)
# Apply configuration
sudo exportfs -ra
sudo systemctl restart nfs-kernel-server

# Check exports
sudo exportfs -v

NFS Client Configuration

# Install NFS client
sudo apt install nfs-common

# Create mount point
sudo mkdir /mnt/nfs-share

# Mount NFS share
sudo mount -t nfs 192.168.1.10:/srv/nfs/public /mnt/nfs-share
# /etc/fstab (permanent mount)
192.168.1.10:/srv/nfs/public    /mnt/nfs-share    nfs    defaults    0    0

NFS Security Notes

• NFS v3 tidak encrypted by default

• Gunakan NFS v4 dengan Kerberos untuk security

• Restrict access menggunakan firewall rules

File Permissions & Access Control

Samba Permissions

# Advanced Samba Share
[finance]
    comment = Finance Department
    path = /srv/samba/finance
    valid users = @finance-group
    read only = no
    create mask = 0660
    directory mask = 0770
    force group = finance-group

[public]
    comment = Public Share
    path = /srv/samba/public
    guest ok = yes
    read only = yes
    browseable = yes
# Create groups & users
sudo groupadd finance-group
sudo usermod -aG finance-group alice
sudo usermod -aG finance-group bob

# Set directory permissions
sudo chgrp finance-group /srv/samba/finance
sudo chmod 770 /srv/samba/finance

Advanced Access Control

1
User-based Access
valid users = alice, bob
invalid users = guest
2
Group-based Access
valid users = @admin-group
write list = @editors
3
IP-based Restriction
hosts allow = 192.168.1.0/24
hosts deny = ALL

Security Best Practices

• Disable guest access untuk sensitive data

• Gunakan strong passwords untuk Samba users

• Implement firewall rules

• Regular audit access logs

Disk Quota Management

Praktik: Implementasi Disk Quotas

Enable Quota System

# Edit /etc/fstab
/dev/sda1 /home ext4 defaults,usrquota,grpquota 0 2

# Remount filesystem
sudo mount -o remount /home

# Create quota files
sudo quotacheck -cum /home
sudo quotaon /home
1
Check Quota Status
sudo quotacheck -avum
sudo repquota -a
2
Set User Quotas
sudo edquota -u alice
sudo setquota -u alice 100M 150M 0 0 /home

Quota Configuration

User Soft Limit Hard Limit Current Usage
alice 100 MB 150 MB 45 MB
bob 200 MB 250 MB 180 MB
charlie 50 MB 75 MB 65 MB 
# Group quotas
sudo edquota -g students
sudo setquota -g students 1G 2G 0 0 /home

# Copy quotas
sudo edquota -p alice -u bob
sudo edquota -p alice -u charlie
Soft vs Hard Limits:
• Soft Limit: Warning threshold
• Hard Limit: Absolute maximum
• Grace Period: Time to comply after soft limit

Quota Management Script

#!/bin/bash
# quota-monitor.sh

# Check quota usage
repquota -a | while read line; do
    user=$(echo $line | awk '{print $1}')
    used=$(echo $line | awk '{print $3}')
    soft=$(echo $line | awk '{print $4}')
    
    if [ "$used" -gt "$soft" ] 2>/dev/null; then
        echo "WARNING: $user exceeded quota"
        echo "$used KB used, limit: $soft KB"
        
        # Send email notification
        mail -s "Quota Warning" $user@domain.com << EOF
Your disk quota has been exceeded.
Current usage: $used KB
Limit: $soft KB
Please clean up your files.
EOF
    fi
done
# Setup cron job for monitoring
sudo crontab -e

# Run every hour
0 * * * * /usr/local/bin/quota-monitor.sh
Backup Strategies

Data Protection & Disaster Recovery

Types of Backup

Full Backup

Pros: Complete data copy, fastest restore

Cons: Large storage, longer backup time

Frequency: Weekly/Monthly

Incremental Backup

Pros: Fast backup, minimal storage

Cons: Complex restore process

Frequency: Daily

Differential Backup

Pros: Balance between full and incremental

Cons: Growing backup size

Frequency: Daily/Weekly

3-2-1 Backup Rule

3-2-1

3 copies of important data

2 different storage media

1 offsite backup copy

Backup Schedule Example

Daily: Incremental backup (automated)

Weekly: Differential backup

Monthly: Full backup

Quarterly: Archive to offsite storage

Backup Implementation

Rsync Backup

# Basic rsync backup
rsync -avz /srv/samba/ /backup/daily/

# Remote backup
rsync -avz /srv/samba/ user@backup-server:/backup/

# Incremental with hard links
rsync -avz --link-dest=/backup/previous/ 
      /srv/samba/ /backup/current/
1
Create Backup Script
Automated backup dengan rotation policy
2
Schedule with Cron
0 2 * * * /usr/local/bin/backup.sh

Advanced Backup Script

#!/bin/bash
# backup.sh - Automated backup script

BACKUP_SOURCE="/srv/samba"
BACKUP_DEST="/backup"
DATE=$(date +%Y%m%d_%H%M%S)
RETENTION_DAYS=30

# Create backup directory
mkdir -p $BACKUP_DEST/$DATE

# Perform backup
rsync -avz --delete \
      --link-dest=$BACKUP_DEST/latest \
      $BACKUP_SOURCE/ \
      $BACKUP_DEST/$DATE/

# Update latest symlink
rm -f $BACKUP_DEST/latest
ln -s $DATE $BACKUP_DEST/latest

# Cleanup old backups
find $BACKUP_DEST -maxdepth 1 -type d \
     -mtime +$RETENTION_DAYS -exec rm -rf {} \;

# Log backup completion
echo "$(date): Backup completed successfully" >> /var/log/backup.log
Restore Procedures

Testing Restore Procedures

Restore Types

1
Full System Restore
Complete recovery dari total system failure
2
Selective File Restore
Restore specific files atau directories
3
Point-in-Time Recovery
Restore ke waktu tertentu sebelum masalah terjadi 
# Full restore
rsync -avz /backup/20241201_020000/ /srv/samba/

# Selective restore
rsync -avz /backup/latest/important_files/ 
      /srv/samba/important_files/

# Verify restore
diff -r /backup/latest/ /srv/samba/

Tuning dan Monitoring File Server

Performance Tuning

# Samba Performance Tuning
[global]
    socket options = TCP_NODELAY IPTOS_LOWDELAY
    read raw = yes
    write raw = yes
    max xmit = 65535
    dead time = 15
    getwd cache = yes
1
Network Optimization
• Jumbo frames untuk Gigabit networks
• NIC teaming untuk bandwidth aggregation
2
Storage Optimization
• SSD untuk metadata dan cache
• RAID configuration sesuai workload
3
Memory Optimization
• Increase file system cache
• Optimize Samba memory usage

Monitoring & Alerting

# Monitor file server performance
iostat -x 1       # Disk I/O statistics
iftop             # Network traffic monitoring
smbstatus         # Samba connections
netstat -i        # Network interface statistics

# Check file server health
df -h             # Disk usage
free -m           # Memory usage
uptime            # System load

Key Metrics to Monitor

• Throughput: MB/s transfer rates

• Latency: Response time per request

• Connections: Concurrent user sessions

• Storage: Disk usage dan IOPS

• Network: Bandwidth utilization

Alert Thresholds

• Disk usage > 80%

• Memory usage > 85%

• Network utilization > 70%

• Response time > 5 seconds
