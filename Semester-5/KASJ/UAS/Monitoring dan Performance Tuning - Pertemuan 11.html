Windows Performance Monitoring
Using Built-in Tools

Performance Monitor (PerfMon)

# Launch Performance Monitor
perfmon.exe

# Key Performance Counters:
    - Processor(_Total)\% Processor Time
    - Memory\Available MBytes  
    - PhysicalDisk(_Total)\Avg. Disk Queue Length
    - Network Interface\Bytes Total/sec
Pro Tip: Create custom Data Collector Sets for automated monitoring and logging

PowerShell Monitoring

# Get CPU Usage
Get-Counter "\Processor(_Total)\% Processor Time"

# Memory Information
Get-WmiObject -Class Win32_OperatingSystem |
    Select TotalVisibleMemorySize, FreePhysicalMemory

# Disk Space
Get-WmiObject -Class Win32_LogicalDisk |
    Where-Object {$_.DriveType -eq 3} |
    Select DeviceID, Size, FreeSpace
Automation: Schedule PowerShell scripts for regular monitoring reports
Linux Performance Monitoring
Command Line Tools

Essential Commands

# System Overview
htop
top

# CPU Information
cat /proc/cpuinfo
lscpu

# Memory Usage
free -h
cat /proc/meminfo

# Disk Usage
df -h
du -sh /path/to/directory

# Network Statistics
netstat -tuln
ss -tuln

Advanced Monitoring

# I/O Statistics
iostat -x 1

# Network Interface Statistics
iftop
netstat -i

# Process Monitoring
ps aux --sort=-%cpu | head
ps aux --sort=-%mem | head

# System Load
uptime
w

# Real-time System Calls
strace -p PID
Note: Some tools may need installation: apt-get install htop iotop iftop

Common Performance Bottlenecks
Identification and Analysis
Bottleneck Type Symptoms Monitoring Tools Quick Solutions
CPU Bottleneck High CPU usage (>80%), slow response times PerfMon, htop, Task Manager Process optimization, CPU upgrade, load balancing
Memory Bottleneck High memory usage, excessive paging Memory counters, free command Memory upgrade, application optimization
Disk I/O Bottleneck High disk queue length, slow file operations iostat, Disk Performance counters SSD upgrade, disk optimization, RAID
Network Bottleneck High network utilization, packet loss netstat, Network Monitor, iftop Bandwidth upgrade, network optimization
Application Bottleneck Slow queries, memory leaks, inefficient code Application-specific tools, profilers Code optimization, database tuning

Case Study: Performance Problem Solving
Real-world Scenario Analysis

Scenario: E-commerce Website Slowdown

Sebuah website e-commerce mengalami perlambatan significant selama jam sibuk (19:00-22:00). Customer complain response time lambat dan beberapa transaksi gagal.

Investigation Steps

1
Data Collection
Kumpulkan metrics: CPU, Memory, Disk I/O, Network, Application logs
2
Pattern Analysis
Analisis pola: Kapan masalah terjadi? Resource mana yang bottleneck?
3
Root Cause Analysis
Identifikasi penyebab: Database query lambat, insufficient memory

Solution Implementation

Solutions Applied

  • Database Optimization: Index optimization, query tuning
  • Caching Implementation: Redis cache for frequently accessed data
  • Load Balancing: Distribute traffic across multiple servers
  • Resource Scaling: Increase memory and CPU capacity

Results

Response Time: Improved from 8s to 2s
Success Rate: Increased from 85% to 99.5%
User Satisfaction: Significant improvement in user experience
