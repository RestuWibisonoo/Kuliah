Log Analysis Fundamentals
Understanding Network and System Logs
Log Analysis Essentials

System Logs

  • /var/log/syslog
  • /var/log/messages
  • Windows Event Logs
  • Application logs

Network Logs

  • Router/Switch logs
  • Firewall logs
  • DHCP logs
  • DNS query logs

Security Logs

  • Authentication logs
  • Access logs
  • IDS/IPS alerts
  • VPN connection logs

Log Analysis Commands

# View recent logs tail -f /var/log/syslog tail -n 100 /var/log/messages # Search patterns grep "error" /var/log/syslog grep -i "failed" /var/log/auth.log # Advanced filtering awk '/ERROR/ {print $1,$2,$3}' log.txt sed -n '100,200p' /var/log/syslog

Log Analysis Tips

  • Timestamp correlation: Cocokkan waktu kejadian
  • Pattern recognition: Cari pola berulang
  • Severity levels: Fokus pada ERROR/CRITICAL
  • Context analysis: Lihat log sebelum/sesudah
  • Cross-reference: Bandingkan multiple log sources
Time Synchronization: Pastikan semua device memiliki waktu yang sinkron untuk analisis yang akurat!

Common Network Problems
Frequent Issues and Symptoms
Top Network Issues

Connectivity Issues

Symptoms: Cannot reach host, timeouts

Causes: Cable issues, interface down, wrong IP config

Performance Issues

Symptoms: Slow response, high latency

Causes: Bandwidth saturation, congestion, routing loops

DNS Problems

Symptoms: Cannot resolve hostnames

Causes: DNS server down, wrong configuration, cache issues

DHCP Issues

Symptoms: No IP address assignment

Causes: DHCP server down, scope exhaustion, VLAN issues

Routing Problems

Symptoms: Cannot reach remote networks

Causes: Missing routes, routing loops, metric issues

Security Blocks

Symptoms: Connection refused, filtered

Causes: Firewall rules, ACLs, port security

Troubleshooting Priority:
  1. Physical Layer (cables, interfaces)
  2. Data Link Layer (switching, VLANs)
  3. Network Layer (IP, routing)
  4. Application Layer (services, protocols)

Bandwidth and Latency Problems
Performance Analysis

Latency Issues

  • High ping times
  • Packet loss
  • Jitter variations
  • Buffer delays

Throughput Issues

  • Bandwidth saturation
  • Duplex mismatches
  • Queue congestion
  • Protocol overhead

Application Issues

  • TCP window scaling
  • SSL/TLS overhead
  • Database queries
  • Server processing

Performance Testing Tools

# Bandwidth testing iperf3 -c server-ip iperf3 -s (server mode) # Extended ping test ping -c 100 -i 0.2 target ping -s 1472 target (MTU test) # Monitor interface stats watch -n 1 "cat /proc/net/dev" netstat -i

Performance Metrics

Metric Good Poor
Latency (LAN) < 1ms > 10ms
Latency (WAN) < 50ms > 200ms
Packet Loss < 0.1% > 1%
Jitter < 5ms > 50ms
Common Causes: Bandwidth saturation, duplex mismatch, excessive collisions, routing inefficiencies

DNS Troubleshooting
Domain Name Resolution Issues
DNS Diagnosis & Resolution

Common DNS Symptoms

  • Cannot resolve hostnames to IP addresses
  • Slow website loading despite good connectivity
  • Intermittent resolution failures
  • "Server not found" errors
  • Wrong IP resolution for known domains

DNS Testing Commands

# Basic DNS lookup nslookup google.com dig google.com # Specific DNS server nslookup google.com 8.8.8.8 dig @1.1.1.1 google.com # Reverse DNS nslookup 8.8.8.8 dig -x 8.8.8.8

Advanced DNS Diagnosis

# Check DNS configuration cat /etc/resolv.conf (Linux) ipconfig /all (Windows) # Flush DNS cache ipconfig /flushdns (Windows) sudo systemctl flush-dns (Linux) # Trace DNS resolution dig +trace google.com nslookup -debug google.com

DNS Solutions

  • Check DNS server: Verify server availability
  • Try alternative DNS: 8.8.8.8, 1.1.1.1, 9.9.9.9
  • Clear cache: Flush local DNS cache
  • Check firewall: Allow port 53 UDP/TCP
  • Test with IP: Bypass DNS entirely
DNS Hierarchy Test: Test from specific → general: Local DNS → ISP DNS → Public DNS (8.8.8.8) → Root servers
Quick DNS Test: ping domain-name vs ping IP-address - jika IP works tapi domain tidak, masalahnya di DNS!

Client-Side Diagnosis

# Release and renew IP ipconfig /release ipconfig /renew # Check current config ipconfig /all ip addr show # Check DHCP lease cat /var/lib/dhcp/dhclient.leases # Manual DHCP request dhclient -v eth0

Common DHCP Problems

  • No IP address assigned (169.254.x.x)
  • Wrong subnet assignment
  • Duplicate IP conflicts
  • Lease expiration issues

Server-Side Diagnosis

# Check DHCP service status systemctl status dhcpd service dhcp status # View DHCP logs tail -f /var/log/dhcp.log journalctl -u dhcpd -f # Check scope utilization dhcp-lease-list show ip dhcp pool (Cisco) # Verify configuration dhcpd -t -cf /etc/dhcp/dhcpd.conf

DHCP Solutions

  • Expand address pool
  • Check DHCP relay configuration
  • Verify VLAN settings
  • Test with static IP first
Issue Symptom Likely Cause Solution
No IP 169.254.x.x address DHCP server unreachable Check network, server status
Wrong subnet IP in wrong range Multiple DHCP servers Identify rogue DHCP
IP conflict Duplicate address error Static IP overlap Check reservations, exclusions

Wireshark for Network Analysis
Deep Packet Inspection and Protocol Analysis
Packet-Level Troubleshooting

Capture Traffic

Monitor real-time network packets

Apply Filters

Focus on relevant traffic only

Analyze Protocols

Deep dive into protocol behavior

Common Wireshark Filters

# Protocol filters http tcp udp dns dhcp arp # IP address filters ip.addr == 192.168.1.1 ip.src == 10.0.0.1 ip.dst == 8.8.8.8 # Port filters tcp.port == 80 udp.port == 53

Troubleshooting with Wireshark

  • TCP Analysis: Retransmissions, timeouts, window size
  • DNS Issues: Query/response matching, errors
  • HTTP Problems: Status codes, response times
  • Performance: Bandwidth usage, latency patterns

Analysis Features

  • Flow Analysis: Follow TCP streams
  • Statistics: Protocol hierarchy, conversations
  • Expert Analysis: Automatic problem detection
  • Time Analysis: Response time graphs

Best Practices

  • Capture close to problem: Minimize noise
  • Use appropriate filters: Focus analysis
  • Save captures: For later analysis
  • Compare baseline: Normal vs problem traffic
Pro Tip: Start with basic filters (host, protocol, port) then narrow down based on findings. Use "Follow TCP Stream" for application-level issues.

Documentation & Best Practices
Effective Troubleshooting Habits
Professional Troubleshooting Standards

Documentation Requirements

Problem Record Template

  • Date/Time: When problem occurred
  • Symptoms: What users experienced
  • Scope: Affected systems/users
  • Impact: Business/operational impact
  • Root Cause: Identified cause
  • Solution: Steps taken to resolve
  • Prevention: How to avoid recurrence

Best Practices

Troubleshooting Excellence

  • Stay systematic: Don't skip steps
  • Document everything: Commands, outputs, changes
  • Test incrementally: One change at a time
  • Backup configurations: Before making changes
  • Verify solutions: Ensure complete resolution
  • Share knowledge: Update team documentation

Time Management

Set reasonable timeframes for troubleshooting phases

Communication

Keep stakeholders informed of progress and ETA

Change Control

Follow change management procedures for fixes

# Sample troubleshooting log entry Date: 2024-01-15 14:30 Problem: Users cannot access internal web server Symptoms: HTTP timeouts, connection refused errors Affected: Marketing department (20 users) Investigation: 14:35 - Verified web server service running 14:40 - Checked firewall rules - FOUND: Port 80 blocked 14:45 - Added firewall exception for port 80 14:50 - Verified user access restored Resolution: Firewall rule added to allow HTTP traffic Root Cause: Recent security policy blocked web traffic Prevention: Review firewall rules before policy deployment
Knowledge Base: Maintain searchable database of problems and solutions for faster future resolution.
