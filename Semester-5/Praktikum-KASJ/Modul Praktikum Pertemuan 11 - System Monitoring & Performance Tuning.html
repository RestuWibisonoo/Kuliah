Dasar Teori

1. System Monitoring Overview

System Monitoring adalah proses berkelanjutan untuk mengawasi, mengukur, dan menganalisis performa sistem komputer dan infrastruktur IT. Monitoring mencakup pengumpulan data tentang resource usage (CPU, memory, disk, network), application performance, dan system health. Tujuan utama monitoring adalah mendeteksi masalah sebelum berdampak pada user, mengoptimalkan resource allocation, dan memastikan sistem berjalan dengan performa optimal.

2. Performance Counters & Metrics

Performance counters adalah nilai numerik yang merepresentasikan aspek spesifik dari system performance. Metrics yang umum dimonitor meliputi: CPU utilization, memory usage, disk I/O, network throughput, process count, dan response time. Key Performance Indicators (KPIs) ini membantu administrator memahami system behavior, identify bottlenecks, dan membuat keputusan tentang capacity planning dan optimization.

3. Monitoring Tools Ecosystem

Ekosistem monitoring tools di Linux sangat beragam, mulai dari command-line utilities sederhana seperti top dan vmstat, hingga sophisticated monitoring platforms seperti Prometheus dan Grafana. Setiap tool memiliki keunggulan spesifik: real-time monitoring, historical analysis, alerting capabilities, atau visualization features. Pemilihan tool yang tepat bergantung pada requirements, scale, dan complexity dari infrastruktur yang dimonitor.

4. Alert Configuration & Threshold Management

Alert configuration adalah proses mendefinisikan conditions yang memicu notifikasi kepada administrator. Effective alerting memerlukan careful threshold setting untuk menghindari alert fatigue (terlalu banyak false positives) atau missed incidents (thresholds terlalu tinggi). Alert system yang baik menggunakan multiple severity levels, escalation policies, dan intelligent notification routing untuk memastikan right people mendapat right information pada right time.

System Resource Monitoring

CPU Monitoring

Penjelasan: CPU (Central Processing Unit) adalah otak dari komputer yang menjalankan instructions. Monitoring CPU usage penting untuk memahami computational workload dan mendeteksi process yang consume excessive resources. CPU metrics meliputi utilization percentage, load average, context switches, dan per-core statistics.

Key Metrics:

  • CPU Utilization: Persentase waktu CPU actively processing instructions
  • Load Average: Average number of processes waiting for CPU time (1, 5, 15 minutes)
  • Context Switches: Frequency of CPU switching between processes
  • Idle Time: Persentase waktu CPU tidak digunakan
# ======================================================================== # TOP - Interactive Process Viewer # ======================================================================== top # Real-time system monitoring tool # Shows: CPU usage, memory usage, running processes # Interactive commands: # - Press 'q' to quit # - Press 'k' to kill a process # - Press 'r' to renice a process # - Press '1' to show individual CPU cores # - Press 'shift+m' to sort by memory usage # - Press 'shift+p' to sort by CPU usage top -b -n 1 # Batch mode - F1: Help, F2: Setup, F3: Search # - F4: Filter, F5: Tree view, F6: Sort by # - F9: Kill process, F10: Quit htop -u username # Show processes for specific user # Filter by username htop -p PID,PID,PID # Monitor specific processes # Comma-separated PIDs # Example: htop -p 1234,5678,9012 # ======================================================================== # UPTIME - System Load Average # ======================================================================== uptime # Show: current time, uptime, users, load average # Output: 10:23:45 up 5 days, 3:21, 2 users, load average: 0.5, 0.7, 0.8 # Load average: 1min, 5min, 15min # Interpretation: # - Load < number of cores = system not overloaded # - Load = number of cores = system at capacity # - Load > number of cores = system overloaded # Check number of CPU cores nproc # OR lscpu | grep "^CPU(s):" # ======================================================================== # MPSTAT - Multi-Processor Statistics # ======================================================================== sudo apt install sysstat # Install if not available mpstat # Report CPU statistics # Shows utilization per CPU # %usr, %sys, %idle, %iowait mpstat -P ALL # Show statistics for ALL CPUs individually # -P ALL = all processors # Useful untuk identify CPU imbalance mpstat -P ALL 2 5 # Report every 2 seconds, 5 times # Continuous monitoring # Format: mpstat [options] [interval] [count] mpstat 1 # Report every 1 second indefinitely # Real-time CPU monitoring # Press Ctrl+C to stop # ======================================================================== # PIDSTAT - Process Statistics # ======================================================================== pidstat # Report per-process CPU usage # Shows: PID, %usr, %system, %CPU, Command # Default: active processes only pidstat -p PID # Monitor specific process # Track single process CPU usage pidstat -u 2 5 # CPU statistics every 2 seconds, 5 times # -u = CPU utilization pidstat -r # Memory usage statistics per process # Shows: minflt/s, majflt/s, VSZ, RSS pidstat -d # Disk I/O statistics per process # Shows: kB_rd/s, kB_wr/s # ======================================================================== # PS - Process Status # ======================================================================== ps aux # List all processes with detailed info # a = all users, u = user-oriented format, x = include processes without tty # Shows: USER, PID, %CPU, %MEM, VSZ, RSS, TTY, STAT, START, TIME, COMMAND ps aux --sort=-%cpu | head -10 # Top 10 CPU-consuming processes # --sort=-%cpu = sort by CPU descending # head -10 = show first 10 lines ps aux --sort=-%mem | head -10 # Top 10 memory-consuming processes # Identify memory hogs ps -ef # Full format listing # Shows: UID, PID, PPID, C, STIME, TTY, TIME, CMD # -e = all processes, -f = full format ps -eo pid,ppid,cmd,%mem,%cpu --sort=-%cpu | head # Custom output format # -o = user-defined format # Show specific columns only # ======================================================================== # CPU FREQUENCY & GOVERNOR # ======================================================================== lscpu # Display CPU architecture information # Shows: cores, threads, frequency, cache sizes # Architecture, CPU op-mode(s), vendor, model cat /proc/cpuinfo # Detailed CPU information # Shows per-core details # Processor, vendor_id, cpu MHz, cache size cpupower frequency-info # Show current CPU frequency settings # Requires cpupower package # sudo apt install linux-tools-common cat /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor # Check CPU governor for each core # Common governors: performance, powersave, ondemand, conservative # Set CPU governor (requires root) echo "performance" | sudo tee /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor # Set all cores to performance mode # Maximize performance (higher power consumption)

Memory Monitoring

Penjelasan: Memory (RAM) adalah temporary storage yang digunakan untuk menjalankan programs dan storing data yang sedang diproses. Monitoring memory usage critical untuk memastikan sistem tidak kehabisan RAM yang dapat menyebabkan swapping (menggunakan disk sebagai RAM) dan performance degradation. Memory metrics meliputi total, used, free, available, buffers, cached, dan swap usage.

Memory Types:

  • Used: Memory actively used by processes
  • Free: Completely unused memory
  • Available: Memory available for new processes (includes cache yang bisa di-reclaim)
  • Buffers/Cache: Memory used for disk caching (can be freed if needed)
  • Swap: Disk space used as virtual memory when RAM is full
# ======================================================================== # FREE - Memory Usage # ======================================================================== free # Display memory usage # Shows: total, used, free, shared, buffers, cache, available # Output in kilobytes by default free -h # Human-readable format # Shows in GB/MB automatically # -h = human readable # Easier to read: "7.8G" instead of "8154200" free -m # Output in megabytes # -m = megabytes # More precise than -h for scripting free -g # Output in gigabytes # -g = gigabytes free -s 2 # Update every 2 seconds # -s = seconds interval # Continuous memory monitoring # Press Ctrl+C to stop watch -n 1 free -h # Update every 1 second with full screen refresh # watch command for continuous monitoring # Better visualization than free -s # ======================================================================== # VMSTAT - Virtual Memory Statistics # ======================================================================== vmstat # Report virtual memory statistics # Shows: processes, memory, swap, io, system, cpu # Single snapshot vmstat 2 5 # Report every 2 seconds, 5 times # Format: vmstat [interval] [count] # Continuous monitoring vmstat -a # Active/inactive memory # -a = show active and inactive memory # Breakdown of memory usage vmstat -s # Memory statistics in table format # -s = summary statistics # Detailed memory breakdown vmstat -d # Disk statistics # -d = disk stats # Shows: reads, writes, I/O vmstat -w # Wide output mode # -w = wide mode # Better formatting for modern terminals # ======================================================================== # SMEM - Memory Reporting Tool # ======================================================================== sudo apt install smem smem # Report memory usage by process # Shows: PID, User, Command, Swap, USS, PSS, RSS # USS = Unique Set Size (memory unique to process) # PSS = Proportional Set Size (shared memory divided) # RSS = Resident Set Size (total memory in RAM) smem -t # Show totals at bottom # -t = totals # Summary of all processes smem -u # Group by user # -u = by user # See which user consuming most memory smem -s uss # Sort by USS (unique memory) # -s = sort by column # Most accurate measure of process memory usage smem -p # Show memory usage as percentage # -p = percentage # Easier to understand proportions # ======================================================================== # SLABTOP - Kernel Slab Cache # ======================================================================== sudo slabtop # Display kernel slab cache information # Shows: cache name, objects, active, size # Real-time monitoring of kernel memory # Press 'q' to quit sudo slabtop -o # Display once and exit # -o = once # For scripting # ======================================================================== # /PROC/MEMINFO - Detailed Memory Info # ======================================================================== cat /proc/meminfo # Detailed memory statistics # Shows: MemTotal, MemFree, MemAvailable, Buffers, Cached # Comprehensive memory breakdown grep -E "MemTotal|MemFree|MemAvailable|Cached" /proc/meminfo # Filter specific memory metrics # Extract key information # Useful for scripting # Calculate used memory percentage awk '/MemTotal/ {total=$2} /MemAvailable/ {avail=$2} END {print "Used: " (100-avail/total*100) "%"}' /proc/meminfo # Calculate percentage from /proc/meminfo # More accurate than simple calculations # ======================================================================== # OOM (Out of Memory) Monitoring # ======================================================================== dmesg | grep -i "out of memory" # Check for OOM killer events # OOM killer terminates processes when memory exhausted # Important for troubleshooting crashes dmesg | grep -i "killed process" # See which processes were killed by OOM # Identify problematic applications grep -i "oom" /var/log/syslog # Check syslog for OOM events # Alternative log location cat /proc/sys/vm/overcommit_memory # Check memory overcommit setting # 0 = heuristic (default) # 1 = always overcommit # 2 = don't overcommit # Adjust swappiness (how aggressively use swap) cat /proc/sys/vm/swappiness # Check current swappiness value # Default: 60 # Lower = less swap usage, higher = more swap usage sudo sysctl vm.swappiness=10 # Set swappiness to 10 (less swapping) # Temporary change (until reboot) # Permanent change echo "vm.swappiness=10" | sudo tee -a /etc/sysctl.conf sudo sysctl -p # Add to sysctl.conf for persistence

Disk I/O Monitoring

Penjelasan: Disk I/O monitoring mengukur read/write operations pada storage devices. Disk bottlenecks dapat significantly impact overall system performance karena disk adalah komponen paling lambat dalam sistem. Monitoring disk I/O membantu identify slow disks, I/O contention, dan optimization opportunities.

# ======================================================================== # IOSTAT - I/O Statistics # ======================================================================== sudo apt install sysstat # Install if not available iostat # Report CPU and I/O statistics # Shows: device utilization, transfer rates # Single snapshot iostat -x # Extended statistics # -x = extended display # Shows: r/s, w/s, rkB/s, wkB/s, await, %util # More detailed disk metrics iostat -x 2 5 # Report every 2 seconds, 5 times # Continuous I/O monitoring iostat -d # Display only device statistics # -d = devices only (skip CPU) # Focus on disk metrics iostat -p sda # Show statistics for specific device # -p = partition # Monitor specific disk/partition iostat -x -h # Human-readable output # -h = human readable sizes # Easier to read KB/MB/GB # ======================================================================== # IOTOP - I/O Monitor (like top for disk) # ======================================================================== sudo apt install iotop sudo iotop # Real-time disk I/O monitoring # Shows which processes using disk I/O # Requires root privileges # Interactive display # Columns: TID, PRIO, USER, DISK READ, DISK WRITE, COMMAND sudo iotop -o # Show only processes doing I/O # -o = only active # Filter out idle processes sudo iotop -b -n 3 # Batch mode for 3 iterations # -b = batch mode # -n = number of iterations # Useful for logging sudo iotop -P # Show processes instead of threads # -P = processes # Aggregated view # ======================================================================== # LSBLK - List Block Devices # ======================================================================== lsblk # List all block devices # Shows: NAME, MAJ:MIN, RM, SIZE, RO, TYPE, MOUNTPOINT # Tree view of disks and partitions lsblk -f # Show filesystem information # -f = filesystem # Shows: filesystem type, UUID, mountpoint lsblk -a # List all devices including empty ones # -a = all # ======================================================================== # DF - Disk Free Space # ======================================================================== df # Report filesystem disk space usage # Shows used and available space per filesystem # Output in 1K blocks by default df -h # Human-readable format # -h = human readable # Shows in GB/MB df -h -t ext4 # Show only ext4 filesystems # -t = type # Filter by filesystem type df -i # Show inode usage instead of block usage # -i = inodes # Important: dapat kehabisan inodes meski masih ada space # Each file uses one inode df -h / # Show space for specific mountpoint # Check root filesystem only # ======================================================================== # DU - Disk Usage # ======================================================================== du -sh /var # Show total size of /var directory # -s = summarize (total only) # -h = human readable du -sh /* # Show size of all directories in root # Quick overview of space usage per directory du -h --max-depth=1 /home # Show size of directories in /home (1 level deep) # --max-depth=1 = don't recurse # Identify large directories du -sh /home/* | sort -hr | head -10 # Top 10 largest directories in /home # sort -hr = sort human readable descending # Find space hogs du -a /var/log | sort -nr | head -20 # Find 20 largest files in /var/log # -a = all files (not just directories) # -n = numeric sort # Identify large log files ncdu /home # NCurses Disk Usage - interactive # sudo apt install ncdu # Navigate with arrow keys # Press 'd' to delete # Very useful for cleaning up space # ======================================================================== # SMARTCTL - S.M.A.R.T. Monitoring # ======================================================================== sudo apt install smartmontools sudo smartctl -a /dev/sda # Show all SMART information for /dev/sda # -a = all # Health status, error log, self-test results # Check disk health sudo smartctl -H /dev/sda # Show health status only # -H = health # Quick check: PASSED or FAILED sudo smartctl -t short /dev/sda # Run short self-test # -t = test # Takes a few minutes sudo smartctl -A /dev/sda # Show all attributes # -A = attributes # Temperature, power-on hours, bad sectors, etc.

Network Monitoring

Penjelasan: Network monitoring mengukur data transfer, connection states, bandwidth usage, dan packet statistics. Monitoring network performance penting untuk troubleshooting connectivity issues, identifying bandwidth bottlenecks, dan detecting security anomalies seperti unusual traffic patterns atau port scanning attempts.

# ======================================================================== # SS - Socket Statistics (modern replacement for netstat) # ======================================================================== ss # Display socket statistics # Shows all established connections # Faster than netstat ss -tuln # Show TCP and UDP listening ports # -t = TCP # -u = UDP # -l = listening # -n = numeric (don't resolve names) ss -tulpn # Include process information # -p = process # Shows which program using which port # Requires root for full process info ss -s # Summary statistics # -s = summary # Overview of all connections ss -o state established # Show only established connections # -o = options # Filter by connection state ss dst 192.168.1.100 # Show connections to specific destination # Filter by destination IP ss -t -a # Show all TCP sockets (listening and established) # -a = all # ======================================================================== # NETSTAT - Network Statistics (legacy but still useful) # ======================================================================== netstat -tuln # Show TCP and UDP listening ports # Same flags as ss netstat -i # Show network interfaces statistics # -i = interfaces # RX-OK, TX-OK, RX-ERR, TX-ERR netstat -s # Show protocol statistics # -s = statistics # TCP, UDP, ICMP statistics netstat -r # Show routing table # -r = route # Display kernel routing table netstat -c # Continuous output # -c = continuous # Update every second # ======================================================================== # IFTOP - Network Bandwidth Monitor # ======================================================================== sudo apt install iftop sudo iftop # Real-time network bandwidth usage # Shows which connections using bandwidth # Press 'h' for help # Press 'q' to quit sudo iftop -i eth0 # Monitor specific interface # -i = interface sudo iftop -n # Don't resolve hostnames # -n = numeric # Faster performance sudo iftop -P # Show port numbers # -P = port display # ======================================================================== # NETHOGS - Per-Process Bandwidth Monitor # ======================================================================== sudo apt install nethogs sudo nethogs # Show bandwidth usage per process # Identify which application using network # Real-time monitoring sudo nethogs eth0 # Monitor specific interface sudo nethogs -d 2 # Update delay 2 seconds # -d = delay # ======================================================================== # VNSTAT - Network Traffic Monitor # ======================================================================== sudo apt install vnstat vnstat # Show network traffic statistics # Hourly, daily, monthly stats # Lightweight monitoring daemon vnstat -d # Daily statistics # -d = daily vnstat -m # Monthly statistics # -m = monthly vnstat -l # Live traffic monitoring # -l = live vnstat -i eth0 # Statistics for specific interface # -i = interface # ======================================================================== # TCPDUMP - Packet Analyzer # ======================================================================== sudo tcpdump -i eth0 # Capture packets on eth0 # Real-time packet capture # Press Ctrl+C to stop sudo tcpdump -i eth0 -n # Don't resolve hostnames # -n = numeric # Faster sudo tcpdump -i eth0 port 80 # Capture only HTTP traffic # Filter by port sudo tcpdump -i eth0 host 192.168.1.100 # Capture traffic from/to specific host sudo tcpdump -i eth0 -w capture.pcap # Save capture to file # -w = write # Can be analyzed later with Wireshark sudo tcpdump -i eth0 -c 100 # Capture only 100 packets # -c = count # ======================================================================== # IP - Network Configuration & Statistics # ======================================================================== ip addr show # Show all IP addresses # Display all network interfaces ip -s link # Show network statistics # -s = statistics # RX and TX packets, bytes, errors ip route show # Show routing table # Display all routes ip neigh show # Show ARP table # Neighbor cache (MAC to IP mapping) # ======================================================================== # PING & MTR - Network Connectivity # ======================================================================== ping -c 4 google.com # Send 4 ICMP echo requests # -c = count # Test connectivity and latency ping -i 0.2 -c 10 192.168.1.1 # Ping every 0.2 seconds # -i = interval # Faster testing sudo apt install mtr mtr google.com # Combined ping and traceroute # Real-time network diagnostic # Shows packet loss and latency per hop # Press 'q' to quit mtr -r -c 10 google.com # Report mode (non-interactive) # -r = report # -c 10 = 10 cycles # Suitable for logging

Performance Tuning Basics

Penjelasan: Performance tuning adalah proses mengoptimalkan system configuration untuk achieve better performance, efficiency, atau responsiveness. Tuning melibatkan adjusting kernel parameters, optimizing resource allocation, dan configuring applications untuk maximize performance berdasarkan workload characteristics.

# ======================================================================== # SYSCTL - Kernel Parameter Tuning # ======================================================================== sysctl -a # Show all kernel parameters # -a = all # Hundreds of tunable parameters sysctl vm.swappiness # Show specific parameter value # vm.swappiness controls swap usage sudo sysctl vm.swappiness=10 # Set parameter temporarily # Change until reboot # Permanent changes sudo nano /etc/sysctl.conf # Add: vm.swappiness=10 sudo sysctl -p # Reload sysctl.conf # -p = load from file # Apply permanent changes # ======================================================================== # COMMON PERFORMANCE TUNING PARAMETERS # ======================================================================== # Network tuning sudo sysctl -w net.core.rmem_max=16777216 # Increase receive buffer size # Better for high-speed networks sudo sysctl -w net.core.wmem_max=16777216 # Increase send buffer size sudo sysctl -w net.ipv4.tcp_window_scaling=1 # Enable TCP window scaling # Better throughput for long-distance connections sudo sysctl -w net.ipv4.tcp_timestamps=1 # Enable TCP timestamps # Better round-trip time calculation # File system tuning sudo sysctl -w vm.dirty_ratio=15 # Percentage of memory for dirty pages before sync # Lower = more frequent disk writes sudo sysctl -w vm.dirty_background_ratio=5 # Background writeback threshold # Process tuning sudo sysctl -w kernel.pid_max=65536 # Maximum PID value # Increase if running many processes ulimit -n 65536 # Increase open files limit per process # Default often 1024, too low for busy servers # Permanent ulimit changes in /etc/security/limits.conf echo "* soft nofile 65536" | sudo tee -a /etc/security/limits.conf echo "* hard nofile 65536" | sudo tee -a /etc/security/limits.conf # ======================================================================== # NICE & RENICE - Process Priority # ======================================================================== nice -n 10 command # Run command with lower priority # Nice value: -20 (highest) to 19 (lowest) # Default: 0 # Higher nice = lower priority sudo renice -n -5 -p PID # Change priority of running process # -n = nice value # -p = process ID # Negative values require root ps -eo pid,ni,comm # Show processes with nice values # ni = nice value column # ======================================================================== # IONICE - I/O Priority # ======================================================================== ionice -c 3 command # Run with idle I/O priority # -c = class # Class 1: Real-time, 2: Best-effort, 3: Idle sudo ionice -c 2 -n 0 -p PID # Change I/O priority of running process # -n = priority level (0-7, 0 is highest) # Useful untuk background tasks ionice -p PID # Check current I/O priority # Display class and priority # ======================================================================== # CPU AFFINITY # ======================================================================== taskset -c 0,1 command # Run command on CPU 0 and 1 only # -c = CPU list # Bind process to specific cores taskset -p PID # Show CPU affinity of running process sudo taskset -pc 0-3 PID # Set CPU affinity for running process # Cores 0 through 3 # ======================================================================== # PERFORMANCE PROFILES # ======================================================================== sudo apt install tuned sudo systemctl start tuned sudo systemctl enable tuned tuned-adm list # List available profiles # Profiles: balanced, powersave, throughput-performance, etc. tuned-adm active # Show active profile sudo tuned-adm profile throughput-performance # Switch to performance profile # Optimizes for maximum throughput sudo tuned-adm profile balanced # Balanced profile (default) # Balance between performance and power saving
