CPMK 0901
Kebutuhan Fungsional dan Non-Fungsional
  • Definisi dan perbedaan kebutuhan fungsional vs non-fungsional

  • Identifikasi potensi risiko implementasi

  • Change management dalam implementasi sistem

3Ô∏è‚É£ INFRASTRUKTUR DAN KEAMANAN JARINGAN

20 poin
CPMK 0901
Topologi Jaringan
  • Jenis-jenis topologi jaringan (Star, Ring, Mesh, Hybrid, dll)
  • Kelebihan dan kekurangan masing-masing topologi
  • Pemilihan topologi untuk skala enterprise
  • Scalability dan reliability topologi
Arsitektur Keamanan Berlapis (Defense in Depth)
  • Konsep defense in depth
  • Minimal 6 layer keamanan:
    1. Physical security
    2. Perimeter security (Firewall, IDS/IPS)
    3. Network security (VLAN, Network Segmentation)
    4. Endpoint security (Antivirus, EDR)
    5. Application security (WAF, Input validation)
    6. Data security (Encryption, Access control)
  • Fungsi dan implementasi setiap layer
  • Zero Trust Architecture
Quality of Service (QoS)
  • Konsep dan tujuan QoS
  • Traffic prioritization
  • Bandwidth management
  • Latency dan jitter control
  • Contoh konfigurasi QoS untuk critical system
  • Metrics dan threshold QoS

üí° Tips Belajar:

  • Gambar dan pahami setiap topologi jaringan
  • Buat catatan detail tentang setiap layer keamanan
  • Pelajari contoh konfigurasi QoS praktis

4Ô∏è‚É£ CLOUD COMPUTING DAN INTEGRASI DATA

20 poin
CPMK 1005
Model Layanan Cloud
  • IaaS (Infrastructure as a Service): karakteristik, kasus penggunaan
  • PaaS (Platform as a Service): karakteristik, kasus penggunaan
  • SaaS (Software as a Service): karakteristik, kasus penggunaan
  • Perbandingan dan pemilihan model yang tepat
Deployment Model Cloud
  • Public Cloud: kelebihan, kekurangan, use case
  • Private Cloud: kelebihan, kekurangan, use case
  • Hybrid Cloud: kelebihan, kekurangan, use case
  • Pertimbangan cost, security, scalability, compliance
Strategi Integrasi Data
  • Enterprise Service Bus (ESB)
  • API Management
  • Message Queue (RabbitMQ, Kafka)
  • RESTful API vs SOAP
  • Microservices integration patterns
  • Data synchronization strategies
Auto-scaling dan Load Balancing
  • Konsep horizontal vs vertical scaling
  • Metrics untuk trigger auto-scaling (CPU, Memory, Request Rate)
  • Threshold configuration
  • Load balancing algorithms (Round Robin, Least Connection, IP Hash)
  • Health check mechanisms
  • Disaster recovery dan high availability

üìã Studi Kasus yang Perlu Dipelajari:

  • Migrasi dari on-premise ke cloud
  • Platform e-commerce dengan traffic tinggi
  • Integrasi sistem eksternal (payment gateway, logistics, inventory)

Paradigma Pemrosesan Big Data
  • MapReduce:
    • Konsep map dan reduce

  • Model training dan deployment

Analisis Kompleksitas Algoritma
  • Time complexity (Big O notation)
  • Space complexity
  • Trade-off antara waktu dan ruang
  • Optimisasi algoritma untuk real-time processing
  • Distributed computing optimization
  • Caching strategies

  • Siapkan contoh kasus penggunaan ML di sistem IoT/smart city
