# AWS WAF Rule Example (Terraform)
resource "aws_wafv2_web_acl" "main" {
  name  = "production-waf"
  scope = "REGIONAL"

  default_action {
    allow {}
  }

  # Rate limiting rule
  rule {
    name     = "RateLimitRule"
    priority = 1
    action {
      block {}
    }
    statement {
      rate_based_statement {
        limit              = 2000
        aggregate_key_type = "IP"
      }
    }
    visibility_config {
      cloudwatch_metrics_enabled = true
      metric_name                = "RateLimitRule"
      sampled_requests_enabled   = true
    }
  }

  # SQL injection protection
  rule {
    name     = "SQLiProtection"
    priority = 2
    override_action {
      none {}
    }
    statement {
      managed_rule_group_statement {
        vendor_name = "AWS"
        name        = "AWSManagedRulesSQLiRuleSet"
      }
    }
    visibility_config {
      cloudwatch_metrics_enabled = true
      metric_name                = "SQLiProtection"
      sampled_requests_enabled   = true
    }
  }
}

# Secure API Example (Python Flask)
from flask import Flask, request, jsonify
from functools import wraps
import jwt
from datetime import datetime, timedelta

app = Flask(__name__)
app.config['SECRET_KEY'] = 'your-secret-key'

# JWT Authentication Decorator
def token_required(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        token = request.headers.get('Authorization')

        if not token:
            return jsonify({'message': 'Token missing'}), 401

        try:
            if token.startswith('Bearer '):
                token = token[7:]
            data = jwt.decode(token, app.config['SECRET_KEY'],
                            algorithms=['HS256'])
            current_user = data['user_id']
        except jwt.ExpiredSignatureError:
            return jsonify({'message': 'Token expired'}), 401
        except jwt.InvalidTokenError:
            return jsonify({'message': 'Invalid token'}), 401

        return f(current_user, *args, **kwargs)

    return decorated

# Protected Endpoint
@app.route('/api/data', methods=['GET'])
@token_required
def get_data(current_user):
    return jsonify({
        'message': 'Protected data',
        'user': current_user
    })

üìö Studi Kasus: Capital One Data Breach (2019)

üìã Background

Waktu: Maret - Juli 2019
Target: Capital One Financial Corporation
Data Dicuri: 100 juta aplikasi kartu kredit di AS, 6 juta di Kanada
Kerugian Estimasi: $150-300 juta
Root Cause: Misconfigured Web Application Firewall + Over-permissive IAM

‚è±Ô∏è Timeline Serangan

Step 1: Initial Access

Attacker mengeksploitasi misconfigured WAF yang berjalan di AWS EC2 instance. WAF memiliki vulnerability SSRF (Server-Side Request Forgery).

Step 2: Metadata Service Access

Melalui SSRF, attacker mengakses EC2 Instance Metadata Service (IMDS) di endpoint http://169.254.169.254/latest/meta-data/iam/security-credentials/

Step 3: Credential Theft

Mendapatkan temporary AWS credentials dari IAM role yang attached ke EC2 instance. Role ini memiliki permissions yang terlalu luas (overly permissive).

Step 4: S3 Access & Data Exfiltration

Menggunakan stolen credentials untuk mengakses S3 buckets yang berisi data pelanggan. Download ratusan gigabyte data sensitif tanpa terdeteksi.

üîç Capital One Breach: Root Causes Analysis

‚ùå Misconfigured Firewall

WAF rule yang salah konfigurasi memungkinkan SSRF attacks. Attacker bisa membuat request ke internal services.

‚ùå Over-permissive IAM

EC2 instance memiliki IAM role dengan permission s3:* pada semua resources. Melanggar principle of least privilege.

‚ùå IMDSv1 Usage

Menggunakan Instance Metadata Service versi 1 yang vulnerable terhadap SSRF. IMDSv2 (session-based) lebih secure.

‚ùå Lack of Monitoring

Tidak ada alert untuk unusual S3 access patterns. CloudTrail logs tidak dimonitor dengan baik.

‚ùå No Data Loss Prevention

Tidak ada controls untuk mencegah mass data download. No rate limiting atau anomaly detection.

‚ùå Missing Encryption

Beberapa S3 buckets tidak menggunakan encryption. Data sensitif disimpan dalam plaintext.
